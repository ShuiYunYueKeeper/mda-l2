## 背景与目标

### 背景

MDA 1.0 已完成 L2 命题交付：CLI + Electron GUI、批注内嵌 `.md`、源文件保护、~90% 测试覆盖率、可打包分发。技术底座扎实，但 **不具备直接收费的产品形态**。

2025–2026 年竞品分两类：
- **评审向**（Penmark、md-redline）：选区级批注 + 渲染预览内联高亮 + MCP
- **写作向**（Typora、Obsidian、MarkText）：编辑与预览体验极强，批注/插件为辅

MDA 1.0 的短板不仅是「无 MCP」，更是 **预览与编辑体验未达 ToC 付费门槛**——段落级批注、基础 textarea 编辑器，难以让个人用户为「又一个 Markdown 工具」付钱。

**2026-07-13 用户修订**：
1. **主攻 ToC**（个人消费者付费），企业内购/席位制降为次要或不作为 2.0 主线
2. **核心竞争力 = 批注 + 编辑 + 预览** 三位一体增强；**MCP 重要但非最高优先级**（排在体验打磨之后或并行后半程）

### 目标

1. **产品目标（ToC）**：让个人 AI 开发者愿意 **每天打开 MDA** 写/读 spec 与设计文档——**预览够爽、编辑够顺、批注够准**，形成「写作 → 自审 → 留批注 →（日后）Agent 处理」闭环；体验上对标 Penmark/Typora 的 **日常使用率**，而非仅对标 md-redline 的 MCP 集成度。
2. **商业目标**：Freemium 面向 **个人 Pro** 转化；2.0 发布后 **90 天内获得首笔 ToC 付费**（个人订阅或买断）。
3. **合规目标**：软件著作权登记；MIT 核心与 Pro 模块边界清晰。
4. **工程目标**：1.0 硬约束（源文件保护、批注不可见性、CLI stdout 纯净）不得削弱。

### 成功指标（可核验）

| 维度 | 指标 |
|------|------|
| **预览体验** | 核心场景（长文 spec、含表格/代码/Mermaid/图片）滚动流畅；预览与编辑双向定位误差 ≤ 1 行 |
| **编辑体验** | 大纲跳转、查找替换、同步滚动可用；1MB 文件编辑无明显卡顿 |
| **批注体验** | 支持预览区 **选区批注**（不仅段落级）；批注与选区高亮双向定位 |
| **ToC 付费转化** | 官网/应用内可完成个人 Pro 购买与激活 |
| **MCP（次优先）** | 2.0 内交付基础 MCP tools，与 CLI 语义一致（可排在体验三件套之后） |
| **质量** | `npm test` 全通过；覆盖率 ≥ 1.0 基线（Statements ≥ 88%） |
| **收入** | 发布后 90 天内 ≥ 1 笔 **个人 ToC** 付费 |

## 功能描述（详细）

### F1 — 预览增强（P0 最高）

**目标**：达到 ToC 用户「愿意全屏读长文」的水准，支撑批注锚定与编辑联动。

**候选能力（P1 方案对比后定稿，P0 列需求池）**：

| 编号 | 能力 | 说明 | 建议优先级 |
|------|------|------|------------|
| PV-1 | **编辑↔预览同步滚动** | 滚动一侧，另一侧按比例跟随；点击预览块定位到源码行 | 必做 |
| PV-2 | **文档大纲（TOC）** | 从标题生成可点击大纲，侧栏或浮层 | 必做 |
| PV-3 | **选区高亮与批注锚点** | 预览渲染层标记已批注区间，主题色与级别联动 | 必做 |
| PV-4 | 数学公式 | KaTeX 渲染 `$...$` / `$$...$$` | 强烈建议 |
| PV-5 | 表格增强 | 横向滚动、表头固定或斑马纹 | 建议 |
| PV-6 | 链接预览/外链图标 | 外链安全打开（已有 `openExternal`）+ 可选卡片预览 | 可选 |
| PV-7 | 阅读模式 | 隐藏编辑/批注栏，专注预览 | 可选 |
| PV-8 | 导出 | HTML / PDF 单文件导出 | **Free** |

**约束**：`src/core/renderer.ts` 保持纯净（不注入 data-line）；预览专属逻辑仍在 GUI preload/renderer。

| ED-3 | **Markdown 辅助编辑** | 选区包裹、标题/列表/引用、链接、缩进、移动行等（见下表 ED-A） | 必做 |
| ED-4 | 跳转到行 | `Ctrl+G` 输入行号 | 必做 |
| ED-5 | 编辑器升级评估 | textarea 增强 vs Monaco（**P1 已定：增强 textarea**） | 已决 |
| ED-6 | 可编辑性提示 | 保存状态、未保存批注禁用（1.0 已有，保持） | 保持 |
| ~~ED-7~~ | ~~多文件 / 工作区~~ | 已提升为 **F8 文件管理**（2.0 必做） | — |

### F3 — 批注增强（P0 最高）

**目标**：从「段落级评审工具」升级为「选区级 spec 评审工具」，对齐 Penmark 核心体验，保留 `@anno` Git 友好格式。

| 编号 | 能力 | 说明 | 建议优先级 |
|------|------|------|------------|
| AN-1 | **选区级批注** | 预览或源码选中文本 → 添加批注；JSON 扩展 `anchor` 字段（UTF-16 偏移或等价方案） | 必做 |
| AN-2 | **预览内批注高亮** | 批注区间在预览中可见色块/下划线，点击打开批注卡片 | 必做 |
| AN-3 | 批注↔选区双向定位 | 面板点击批注 → 滚动预览并高亮选区 + 定位源码 | 必做 |
| AN-4 | 保留段落级批注 | 旧数据兼容；无 anchor 的批注仍按段落归属 | 必做 |
| AN-5 | 线程/回复 | 同锚点多条回复 | 2.1 候选 |
| AN-6 | 批注作者字段 | 区分人/Agent | 可选 |

**schema 扩展（向后兼容）**：

```jsonc
{
  "id": "...",
  "content": "...",
  // 新增可选
  "anchor": { "start": 120, "end": 145 }  // UTF-16 offset in source, P1 定精确语义
}
```

**约束**：批注写操作仍走 `@mda/core` writer；扩展 parser 识别 anchor，不破坏源文件保护。

### F8 — 文件管理（P0 最高，ToC 日常写作必备）

**目标**：用户无需离开 MDA 即可完成「新建 spec → 编辑 → 保存 → 切换同目录其他文档」闭环；对标 Typora/Obsidian 的基础文件操作。

**1.0 已有**：`Ctrl+O` 打开单文件、拖拽打开、保存到当前路径、`Ctrl+Shift+O` 在资源管理器中显示当前文件。

| 编号 | 能力 | 说明 | 优先级 |
|------|------|------|--------|
| FM-1 | **新建空白文档** | `Ctrl+N` 打开未命名文档（空内容或可选模板）；标题栏显示「未命名.md」 | 必做 |
| FM-2 | **另存为** | 未命名或 `Ctrl+Shift+S` 弹出保存对话框；扩展名默认 `.md` | 必做 |
| FM-3 | **打开文件** | 保留并增强 `Ctrl+O`；打开后加入最近列表 | 必做 |
| FM-4 | **打开文件夹** | 选择目录作为工作区，侧栏展示该目录下 Markdown 文件树（递归，扩展名同 schema） | 必做 |
| FM-5 | **最近文件** | 菜单「文件 → 最近打开」列表（上限 20）；启动时可恢复上次文件（可选） | 必做 |
| FM-6 | **侧栏文件树** | 打开文件夹后左侧显示可点击树；当前文件高亮；点击切换前走 dirty 确认 | 必做 |
| FM-7 | **欢迎页 / 空态** | 无打开文件时展示：新建 / 打开 / 打开文件夹 / 最近文件 | 必做 |
| FM-8 | 监听文件夹变更 | `fs.watch` 刷新文件树（去抖） | 建议 |
| FM-9 | 多标签页 | 同时打开多个文件 tab | **2.1**，2.0 不做 |

**交互约束**：
- 未保存（dirty）时切换文件/新建/打开：必须确认（沿用 1.0 `guardDiscard`）
- **未落盘文档**（无路径）：禁止批注增删改（与 1.0「无文件禁用批注」一致）；须先另存为
- 最近列表存 `userData/recent-files.json`，路径失效自动剔除

**快捷键（与 1.0 协调）**：

| 操作 | 快捷键 |
|------|--------|
| 新建 | `Ctrl+N` |
| 打开文件 | `Ctrl+O` |
| 打开文件夹 | `Ctrl+Alt+O` |
| 保存 | `Ctrl+S`（无路径时等同另存为） |
| 另存为 | `Ctrl+Shift+S` |
| 在资源管理器中显示 | `Ctrl+Shift+O`（保持 1.0） |

### F4 — Freemium 与 License（P1 定稿，2026-07-13 用户确认）

**分界原则**：**除 AI 相关功能外，2.0 全部列为 Free**；Pro **仅**解锁 AI 编辑助手（BYOK）。

| 能力 | Free | Pro |
|------|------|-----|
| F1 预览增强（含导出 PDF/HTML） | ✓ | ✓ |
| F2 编辑增强（含辅助编辑、Ctrl+Space 补全弹层） | ✓ | ✓ |
| F3 批注增强（选区级） | ✓ | ✓ |
| F8 文件管理 | ✓ | ✓ |
| F6 MCP（全部 tools，含批量/export prompt） | ✓ | ✓ |
| 自动更新（electron-updater） | ✓ | ✓ |
| **F9 AI 续写 / 补全 / 智能美化** | — | ✓（BYOK） |

**交付顺序（硬门禁）**：
1. **Phase A — Free 可交付**：F1+F2+F3+F8+F6+MCP+导出+自动更新 全部完成并通过 AC（不含 AI）
2. **Phase B — Pro AI**：仅在 Phase A **用户验收通过后** 启动 F9

**定价（ToC 假设）**：
- 个人 Pro：**¥68–128/年** 或 **¥199 买断** — **卖点仅为 AI 编辑**
- Free 版功能完整，可长期使用无功能阉割（除 AI）

**License**：离线激活码；Free 安装包无需 License。

### F9 — Pro AI 编辑助手（BYOK）

**目标**：为 Pro 用户提供「在 MDA 内直接 AI 辅助写 spec」的能力，形成相对 Free 工具与纯 IDE 插件的 **明确付费理由**；**MDA 不托管模型、不代收 API 费用**，用户接入自己的 API Key。

| 编号 | 能力 | 说明 | 优先级 |
|------|------|------|--------|
| AI-1 | **API Key 配置** | OpenAI 兼容 + **DeepSeek 预设** + **自定义 Base URL**；Key 仅本地加密 | 必做 |
| AI-2 | **AI 续写** | 光标处或选区后，根据上下文续写 Markdown（显式触发，如 `Ctrl+Shift+Enter`） | 必做 |
| AI-3 | **AI 补全** | **`Ctrl+Space` 弹层预览**短补全；采纳后插入；`Esc` 取消（2.0 不做 inline ghost） | 必做 |
| AI-4 | **AI 智能美化** | 对选区或全文：润色结构/用语/排版（仍输出 Markdown）；**预览 diff** 后用户 **保留原文 / 采纳 AI / 部分编辑** | 必做 |
| AI-5 | 流式输出 | 续写/美化支持 streaming，可中途取消 | 必做 |
| AI-6 | 上下文控制 | 默认携带：当前文件内容（或选区±N 行）、文件名；**不**默认上传其他打开文件 | 必做 |
| AI-7 | 失败与配额提示 | 401/429/网络错误可读提示；不泄露 Key | 必做 |

**交互原则（硬约束）**：
- 所有 AI 产出 **默认不直接覆盖原文**；须经用户 **采纳** 才写入 editor（美化走 diff；续写插入前可预览）
- 未配置有效 API Key 时：Pro 功能入口可见但引导去设置，**Free 用户显示 Pro 升级提示**
- 请求从 **Electron main 进程** 直连用户配置的 Base URL，**不经 MDA 服务器**
- API Key 使用 OS 级安全存储（Electron `safeStorage`）；日志/错误信息 **不得打印 Key**

**明确不做（F9 范围外）**：
- MDA 官方代理 / 打包额度 / 统一计费
- 自动把 AI 输出写入批注行（与 writer 分离；用户可手动加批注）
- 多轮 Agent 自主改文件（2.1；2.0 仅单次续写/美化）

### F5 — 官网与 ToC 获客（P1）

- 落地页主打：**「写 spec 最顺的 Markdown 评审桌面工具」** + **Pro：自带 Key 的 AI 续写与智能美化**
- 演示 GIF：三栏 + 选区批注 + 同步滚动
- 下载 + 个人 Pro 购买（微信/支付宝/爱发电等，P1 选型）
- **不做**企业询价表单为 2.0 主线

### F6 — MCP Server（P2，2.0 内交付但不挡三核心）

独立进程，stdio；tools 与 CLI 语义一致（`scan`/`add`/`edit`/`remove`/`read_file`）。

- **排期**：P4 中放在 F1–F3 体验验收通过 **之后** 启动，或与人并行但不得挤压三核心工期
- **ToC 价值**：付费用户配置 Cursor 后 Agent 能读 `@anno`；是 **留存与传播** 能力，非首屏卖点

### F7 — 软件著作权（并行）

- 登记名称建议：**「MDA Markdown 文档评审系统 V2.0」**（覆盖编辑+预览+批注）
- 主体：个人（ToC 副业常规范畴）；WPS 合规仍建议自行确认

### 明确不做（2.0 范围外）

- ToB 主线：席位 License、SSO、审计、内网部署方案
- 完整 SaaS、实时多人协作
- 替换 `@anno` 语法
- 专利申报（暂缓）
- VS Code 插件（2.0；MCP 可部分替代 IDE 触达）
- **多标签页**同时编辑多文件（2.1）

## 验收标准（Given/When/Then）

- **AC-1（预览）**：Given 含多级标题与 Mermaid 的 `samples/demo.md`，When 打开三栏模式并滚动预览，Then 编辑区同步滚动偏差 ≤ 1 屏行高，且大纲可点击跳转对应源码行。
- **AC-2（编辑·查找）**：Given 已打开 5000+ 行 markdown，When `Ctrl+F` 搜索关键词，Then 高亮全部匹配并支持跳转下一处；`Ctrl+H` 替换可预览后生效。
- **AC-2b（编辑·辅助）**：Given 编辑区选中文字 `hello`，When `Ctrl+B`，Then 选区变为 `**hello**` 且光标位置合理；Given 光标在某行，When `Ctrl+Shift+]` 升标题，Then 行首 `#` 数量正确变化且不破坏围栏内代码。
- **AC-9（文件·新建）**：Given 空窗口，When `Ctrl+N` 并输入内容后 `Ctrl+S`，Then 弹出另存为对话框；保存成功后磁盘出现对应 `.md` 且可添加批注。
- **AC-10（文件·文件夹）**：Given 某目录含 3 个 `.md`，When「打开文件夹」选中该目录，Then 侧栏文件树列出 3 个文件且点击可切换打开。
- **AC-11（文件·最近）**：Given 先后打开 A.md、B.md，When 打开「最近文件」菜单，Then B 在 A 上方；打开 B 后列表更新。
- **AC-3（批注·选区）**：Given 用户在预览中选中文本，When 添加 major 批注，Then 磁盘写入合法 `@anno` 行且含 `anchor`；预览显示对应高亮。
- **AC-4（批注·兼容）**：Given 仅含段落级批注的 1.0 样本，When 用 2.0 打开，Then 批注正常显示、可编辑删除，无需迁移。
- **AC-5（源文件保护）**：Given 选区级批注增删改，When 操作完成，Then 非批注行逐字节不变（`verifySourceProtection` 通过）；注：anchor 存于批注行 JSON 内，不改正文。
- **AC-6（ToC 付费）**：Given 用户完成个人 Pro 购买并配置有效 API Key，When 触发 AI 续写，Then 流式返回内容且仅在接受后写入 editor。
- **AC-6b（AI·美化）**：Given Pro 用户选中一段 Markdown，When「智能美化」，Then 展示原文 vs AI 建议 diff，且提供 **保留 / 采纳 / 取消** 三选一。
- **AC-6c（AI·安全）**：Given 已保存 API Key，When 查看日志或错误提示，Then 不出现 Key 明文；未配置 Key 时调用 AI 仅提示去设置。
- **AC-7（MCP·次优先）**：Given 三核心 AC 已通过，When 启动 MCP 并 `mda_scan`，Then 输出与 `mda-cli scan --format json` 一致。
- **AC-8（质量）**：Given 1.0 测试基线，When `npm test`，Then 全通过且 Statements ≥ 88%。
