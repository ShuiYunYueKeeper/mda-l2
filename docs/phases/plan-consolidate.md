## P4-Step2: 一致性蒸馏 Prompt

```
【必须先执行】读取以下文件并遵循其中的约束：
- agents/cpp-architect.md
- rules/common/wps-cpp-spec/wps-cpp-spec.mdc
- rules/common/wps-cmake-spec/wps-cmake-spec.mdc（涉及 CMakeLists.txt 时）

你的任务是将 Step1 的全部并行提案 **综合分析**：不做「单票当选」，而是以 **一致性矩阵** 为核心，提炼跨提案的趋同（高置信）与分歧（决策焦点），并给出 **初步合成取向**，供后续 Step3 红队与 Step4 终版定稿使用。

输入（全部读取，缺一不可）：
- {task_dir}/thinking/P4-proposal-1.md … P4-proposal-K.md（以任务目录下实际存在的全部 proposal 文件为准；K≥3）
- {task_dir}/thinking/P2-design.md
- {task_dir}/thinking/P1-requirement.md
- 若存在：{task_dir}/thinking/P3-enhanced-context.md

引用纪律（NF-3）：优先用「章节锚点 / 小节标题 / 文件路径」指向源文，避免逐字复述整份设计；提炼表格与矩阵时用短语概括 cells。

## 第一步：通读与决策点抽取

1. 为每个 proposal 摘录：提案编号、相对 P2 推荐方向的 **对齐 / 偏离**、关键权衡与假设（一句话级亦可，但必须可回溯到对应 `P4-proposal-{i}`）。
2. 将讨论收敛到一组 **决策点**（粒度：架构取舍、模块边界、接口形态、风险回滚取向、测试与观测策略等）。决策点数量须覆盖各提案实质差异，避免笼统条目。

## 第二步：一致性矩阵（核心）

为每个决策点建一行（或矩阵块），**列**为各提案 `P4-proposal-{i}`，**单元格**写该提案在该点的立场（选项 / 倾向 / 结论短语）。

对每个决策点追加标注：
- **趋同**：≥2 个提案实质一致（写明一致的具体结论）；若全体实质一致，标注 **全体趋同**。
- **分歧**：存在不可忽略的立场分叉；简要列出分叉选项。

矩阵须可读：宁可多层小表，也不要合并成无法核对的长段落。

## 第三步：趋同点总结（高置信结论）

基于矩阵中标注为 **趋同 / 全体趋同** 的条目：
1. 归纳 **高置信结论**：多条提案独立指向同一方向时，写明结论陈述与 **依据提案编号**。
2. 若趋同建立在 **相同未经证实假设** 上，必须标明 **假设联动风险**（避免虚假一致）。

## 第四步：分歧点分析（低置信 / 决策焦点）

对矩阵中 **分歧** 条目：
1. 列出各方理由（按提案编号或视角归因）。
2. 给出 **综合判断**：不是裁决终稿，而是说明当前证据下更稳妥的中间立场、或明确「证据不足须 Step3 专攻 / 须人工拍板」。
3. 禁止用「采纳提案 X」一言蔽之；若某分歧暂时无解，写入下一步 **未决项**。

## 第五步：初步合成取向

综合各 proposal **可保留优点**（性能路径、可维护抽象、最小改动、可测性、兼容边界等），写一段 **结构化** 的合成框架：
- **保留**：哪些趋同结论直接进入合成底座；
- **取舍**：主要张力轴（例如性能 vs 改动面）上的阶段性取向；
- **显式非目标**：本合成框架刻意不包含的内容（避免与 Step4 终版职责混淆）。

⛔ 本节仍然是「取向与框架」，不得下沉为完整执行任务清单或逐文件 patch 列表。

## 第六步：未决项清单

列出须由 Step3 反方审查重点攻击、或必须由 Step4 / 人工终版定稿的点：
- 每条标注：**议题**、**分歧摘要**、**建议审查角度**（需求缺口 / 耦合风险 / 性能安全 / 测试回滚 / DAG 现实性 等，可与 challenge 角度表对齐）。
- 区分 **可在 consolidated 层澄清** 与 **必须终版或人工** 两类。

## 产出结构与章节标题（必须严格遵守）

写入 **唯一** 产出文件：`{task_dir}/thinking/P4-consolidated.md`

正文 **必须**包含以下二级标题（措辞一致，便于 conductor 巡检）：

1. `## 各提案要点与可保留优点`
   - 逐提案概括要点；「可保留优点」须带来源标注（`P4-proposal-{i}`）。

2. `## 一致性矩阵（趋同与分歧）`
   - 内含完整一致性矩阵及对每条决策点的趋同/分歧标注。
   - 紧随矩阵之后用简短小节归纳：
     - `### 趋同点总结（高置信结论）`
     - `### 分歧点分析（决策焦点）`

3. `## 初步合成取向`

4. `## 未决项与留待终版定稿清单`

```
