# M4 选区批注 — 实机验收清单

> 对应 P3 **M4-6**；验收标准见 [`P0-requirements-v2-commercial.md`](P0-requirements-v2-commercial.md) AC-3、AC-4、AC-5。

```bash
npm run build && npm run gui
```

推荐测试文件：`samples/demo.md`、1.0 样本 `samples/acceptance/A.md`

---

## AC-3（批注·选区）

| 步骤 | 操作 | 预期 |
|------|------|------|
| 1 | 在**预览区**选中一段正文文字，右键「添加选区批注」 | 弹出对话框，显示选中文本摘要 |
| 2 | 填写内容、选 major，保存 | 磁盘批注行 JSON 含 `anchor`（`start`/`end`/`quote`） |
| 3 | 预览区 | 对应文字以级别色高亮显示 |
| 4 | 在**源码区**选中文字，右键添加批注 | 同样写入 `anchor` 并高亮 |
| 5 | 在**代码块/表格**内选区右键添加批注 | 支持写入（围栏内映射到源码 UTF-16）；代码块右键为统一菜单：拷贝 / 拷贝全部 / 添加选区批注 |
| 6 | 点击批注面板该项 | 预览滚动到高亮选区，编辑器选中对应源码范围 |
| 7 | 预览区普通点击（非拖选） | 仍恢复段落级定位，不抢选区焦点 |

## AC-4（批注·兼容）

| 步骤 | 操作 | 预期 |
|------|------|------|
| 1 | 打开 `samples/acceptance/A.md`（仅段落级批注） | 批注正常显示在面板 |
| 2 | 编辑 / 删除旧批注 | 行为与 1.0 一致，无需迁移 |

## AC-5（源文件保护）

| 步骤 | 操作 | 预期 |
|------|------|------|
| 1 | 添加/编辑/删除选区级批注后 | 正文行逐字节不变（仅批注行变化） |
| 2 | `npm test` | writer 源文件保护相关用例通过 |

---

## 锚点失效（orphan）

| 步骤 | 操作 | 预期 |
|------|------|------|
| 1 | 添加选区批注后，修改源码使选区文字变化 | 面板显示「锚点失效」标记 |
| 2 | 预览高亮 | 失效锚点不再高亮 |
