# Prompt 17 — 同文件批量添加批注：行号漂移与沉淀

> 日期：2026-07-20  
> 场景：用 MDA（CLI/MCP）向 `docs/demo/我带AI做工程-MDA人机协作分享.md` 批量挂演讲建议批注  
> 类型：协作翻车沉淀（无产品功能改动）

---

## 用户意图

将 `demo-script.md` 与主文中的演示/口播/操作建议，以批注形式写入主分享稿。

## 遇到的问题

| 现象 | 原因 |
|------|------|
| `未找到第 442 行所属的段落` | ① 并行 `mda_add` 导致行号过期；② `line` 落在空行（不属于任何段落） |
| 管道 `JSON.parse` 失败 | `npm run cli` 横幅混入 stdout，破坏 `--format json` 纯净假设 |
| 后加的批注挂错段 | 上方插入使下方行号整体 +1，仍用「添加前」的行号表 |

## 解决方法

1. **同一文件串行 add**；禁止并行。  
2. **自下而上**（按段落 `startLine` 从大到小），使已算好的上方行号保持有效。  
3. 或每成功一条后 **重新 scan/parse**，按正文关键词再定位。  
4. `line` 只用段落内行（标题/列表/正文 `startLine`），跳过空行。  
5. 读 JSON：MCP `mda_scan` 或 `node dist/cli/main.js scan … --format json`。
