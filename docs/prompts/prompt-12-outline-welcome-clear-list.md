## Prompt 33: 大纲移至预览区左侧并可收起

```
用户: 预览区，大纲放在预览区的左侧，可收起
```

AI 动作：

- `#preview-pane` 横向布局：`outline-host` | splitter | `#preview-scroll`（正文滚动容器）。
- 收起记忆 `localStorage`（`mda-outline-collapsed`）；可拖宽、双击复位。
- 同步滚动改挂 `#preview-scroll`，避免大纲参与正文滚动。

## Prompt 34: 大纲交互对齐参考设计

```
用户: 边界线仅 hover 显示；滚动预览同步高亮标题；收起后展开入口常见；勿遮挡正文；收放勿跳动
```

AI 动作：

- 大纲与预览分隔线默认透明，**hover 大纲区**才显示。
- 滚动侦测 + 按源码行归属标题（`headingLineAtOrBefore`）；编辑定位 / 预览点击均更新大纲高亮。
- 收起后左侧 **44px 窄栏** 放 ≡ 展开按钮（不浮在正文上）；正文 `max-width` 恒定 800，避免收放重排跳动。
- 「要点」Tab / AI 总结暂不介入。

## Prompt 35: 清空最近打开与启动起始页

```
用户: 清空最近打开后下次仍打开上次文档；历史为空应显示起始页。
随后纠正：清空最近打开后不应回到起始页。
```

根因：启动恢复工作区时 `refreshWorkspaceTree` 在无当前文件时自动 `requestOpen` 树内第一个 Markdown，覆盖空历史下的欢迎页。

修复：

- 启动仅恢复侧栏工作区，**不**自动开文档；仅用户主动「打开文件夹」且在欢迎态才可开首个文件。
- 清空最近打开：只清列表并通知渲染层刷新；**保持当前文档**；本会话禁止静默 `addRecent`，保证空列表下次启动为起始页。

## Prompt 36: 清空文件列表按钮（不删磁盘文件）

```
用户: 增加清空文件列表按钮（注意不是删除文件）
```

AI 动作：

- 文件侧栏标题栏 **✕**：确认后关闭工作区侧栏、清空树、`setWorkspaceRoot(null)`。
- **不删除**磁盘文件；当前打开文档不受影响。
