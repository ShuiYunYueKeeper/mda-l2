# M2 文件管理 — 实机验收清单

> 对应 P3 **M2-8**；验收标准见 [`P0-requirements-v2-commercial.md`](P0-requirements-v2-commercial.md) AC-9–AC-11。

启动命令（已由 AI 执行）：

```bash
npm run build && npm run gui
```

测试素材目录：`samples/acceptance/`（含 A.md、B.md、C.md）

---

## AC-9（文件·新建）

| 步骤 | 操作 | 预期 |
|------|------|------|
| 1 | 启动后看到欢迎页 | 显示「新建 / 打开 / 打开文件夹」 |
| 2 | `Ctrl+N` | 进入编辑，标题栏显示「未命名.md」 |
| 3 | 输入任意 Markdown 内容 | 预览区实时更新 |
| 4 | `Ctrl+S` | 弹出**另存为**对话框 |
| 5 | 保存为 `.md` 文件 | 磁盘出现该文件 |
| 6 | 点击「+ 添加批注」 | 按钮可用，可成功添加批注 |

**注意**：未保存的未命名文档（UNTITLED）时，批注按钮应**禁用**。

---

## AC-10（文件·文件夹）

| 步骤 | 操作 | 预期 |
|------|------|------|
| 1 | `Ctrl+Alt+O` 或菜单「文件 → 打开文件夹」 | 弹出文件夹选择对话框 |
| 2 | 选择 `samples/acceptance` | 左侧出现文件树 |
| 3 | 树中列出 A.md、B.md、C.md | 共 3 个 Markdown 文件 |
| 4 | 依次点击三个文件 | 预览/批注随文件切换 |

---

## AC-11（文件·最近）

| 步骤 | 操作 | 预期 |
|------|------|------|
| 1 | 菜单「文件 → 打开」，选 `samples/acceptance/A.md` | 文件打开 |
| 2 | 再打开 `samples/acceptance/B.md` | 文件切换 |
| 3 | 菜单「文件 → 最近文件」 | **B.md 在 A.md 上方** |
| 4 | 从最近列表再次打开 B.md | 列表顺序更新（B 仍置顶） |

欢迎页底部「最近打开」列表行为应与菜单一致。

---

## 快捷键速查

| 功能 | 快捷键 |
|------|--------|
| 新建 | `Ctrl+N` |
| 打开文件 | `Ctrl+O` |
| 打开文件夹 | `Ctrl+Alt+O` |
| 保存 | `Ctrl+S`（无路径 → 另存为） |
| 另存为 | `Ctrl+Shift+S` |
| 帮助 | `F1` |
